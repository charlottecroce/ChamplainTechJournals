## Major Steps

### 1. Configure your group’s MLS (MultiLayer Switch)  
  a. Set hostnames to proper name (e.g. “hostname West-MLS”)  
  b. Needs layer 3 (ip routing)  
  c. Define appropriate VLANs (no GUI \- so need to do from CLI)

```
ip routing
vlan 100
name West-Clinic
vlan 110
name West-Admin
vlan 200
name Central-Clinic
vlan 210
name Cenral-Admin
vlan 300
name East-Clinic
vlan 310
name East-Admin
vlan 50
name Backbone
```

  d. Create appropriate VLAN interfaces to act as the default gateway for the Distribution Area VLANs

```
# West-MLS (DG)
interface vlan 100
ip address 192.168.10.1 255.255.255.0
interface vlan 110
ip address 192.168.11.1 255.255.255.0
interface vlan 200
ip address 192.168.20.1 255.255.255.0
interface vlan 210
ip address 192.168.21.1 255.255.255.0
interface vlan 300
ip address 192.168.30.1 255.255.255.0
interface vlan 310
ip address 192.168.31.1 255.255.255.0

# Central-MLS
interface vlan 100
ip address 192.168.10.2 255.255.255.0
interface vlan 110
ip address 192.168.11.2 255.255.255.0
interface vlan 200
ip address 192.168.20.2 255.255.255.0
interface vlan 210
ip address 192.168.21.2 255.255.255.0
interface vlan 300
ip address 192.168.30.2 255.255.255.0
interface vlan 310
ip address 192.168.31.2 255.255.255.0

# East-MLS
interface vlan 100
ip address 192.168.10.3 255.255.255.0
interface vlan 110
ip address 192.168.11.3 255.255.255.0
interface vlan 200
ip address 192.168.20.3 255.255.255.0
interface vlan 210
ip address 192.168.21.3 255.255.255.0
interface vlan 300
ip address 192.168.30.3 255.255.255.0
interface vlan 310
ip address 192.168.31.3 255.255.255.0
```

  e. Configure trunk ports to the edge switches  
    1. Before setting switchport trunk mode, you need to configure the “encapsulation”   
    2. On each trunk port, “switchport trunk encapsulation dot1q”  
      a. This defines how the packets are “tagged”

```
#West-MLS
interface f0/5
switchport trunk encapsulation dot1q
switchport mode trunk
switchport trunk allowed vlan 100,110,200,210,300,310,50
interface range f0/1-2
switchport trunk encapsulation dot1q
switchport mode trunk
switchport trunk allowed vlan 100,110

#Central-MLS
interface f0/5
switchport trunk encapsulation dot1q
switchport mode trunk
switchport trunk allowed vlan 100,110,200,210,300,310,50
interface f0/6
switchport trunk encapsulation dot1q
switchport mode trunk
switchport trunk allowed vlan 100,110,200,210,300,310,50
interface range f0/1-2
switchport trunk encapsulation dot1q
switchport mode trunk
switchport trunk allowed vlan 200,210



#East-MLS
interface f0/5
switchport trunk encapsulation dot1q
switchport mode trunk
switchport trunk allowed vlan 100,110,200,210,300,310,50
interface range f0/1-2
switchport trunk encapsulation dot1q
switchport mode trunk
switchport trunk allowed vlan 300,310
```

  f. Configure RIP

```
router rip
network 192.168.10.0
network 192.168.11.0
network 192.168.20.0
network 192.168.21.0
network 192.168.30.0
network 192.168.31.0
```

  g. Save Configuration  
### 2. Configure North and South edge switches  
  a. Set hostname to proper name (e.g. “hostname North-West-Wing-SW”)  
  b. Define appropriate VLANs  
  c. Assign access ports so that there are interfaces for both VLANs on switch
  d. attach PCs to edge switches (fa0/4)
```
#NW
vlan 100
name West-Clinic
vlan 110
name West-Admin
interface fa0/4
switchport mode access
switchport access vlan 100
#SW
vlan 100
name West-Clinic
vlan 110
name West-Admin
interface fa0/4
switchport mode access
switchport access vlan 110
#NC

name Cenral-Admin

switchport access vlan 200
#SC

name Central-Clinic

name Cenral-Admin
interface fa0/4
switchport mode access
switchport access vlan 210
#NE

name East-Clinic

name East-Admin

switchport mode access
switchport access vlan 300
#SE
vlan 300
name East-Clinic
vlan 310
name East-Admin

switchport access vlan 310
```

  e. Configure Trunk Ports

```

#E

```

  f. Save Configuration

## Tips
Cisco- Save Configuration  
**Configuration changes take effect immediately on Cisco \- but they are not saved\!  If you reboot without saving, the config will be lost.  
To save the current configuration, the command is:
```
copy running-configuration startup-configuration     or  
copy run start 
```

Cisco Config Tips
* “show run” will show your entire configuration  
  * If in configuration mode, “do show run” will do the same thing  
* To delete a configuration setting/line: just add “no” in front and retype the line
